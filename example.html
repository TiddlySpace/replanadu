CSS Box-sizing

This is a nifty global rule you can use to change the way layouts are calculated. CSS originally uses the box-model which uses the stated width of the element as the content section then adds the padding and border on to that. For example:

width: 200px;
padding: 20px; /* each side */
border: 2px; /* each side */
/* equals total width 200+20+20+2+2 = 244 */

Not the most intuitive and something people often get confused with. Wouldn't it be easier if you could set the absolute width and padding and border were subtracted. E.g. the above example would result in a content width of 156px. In fact IE6 does it this way. And now you can do this way again using border-box with the box-sizing rule.

*,
*:after,
*:before {
-webkit-box-sizing: border-box; /* for older webkit versions */
-moz-box-sizing: border-box;
box-sizing: border-box;
}

Supported in all browsers from IE8 up. IE6 already supported it by default so IE7 is the problem. What to do in that? Is IE7 still widely used enough to support it?
